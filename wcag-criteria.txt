WCAG 2.1 Success Criteria

Level A (Must Have):
1.1.1 Non-text Content - Provide text alternatives for non-text content
1.2.1 Audio-only and Video-only (Prerecorded) - Provide alternatives for time-based media
1.2.2 Captions (Prerecorded) - Provide captions for videos
1.2.3 Audio Description or Media Alternative - Provide audio description or text alternative for videos
1.3.1 Info and Relationships - Content structure can be programmatically determined
1.3.2 Meaningful Sequence - Present content in a meaningful order
1.3.3 Sensory Characteristics - Don't rely solely on sensory characteristics
1.4.1 Use of Color - Don't use color as the only means of conveying information
1.4.2 Audio Control - Provide mechanism to stop, pause, or control audio volume
2.1.1 Keyboard - All functionality available from keyboard
2.1.2 No Keyboard Trap - User can navigate in and out with keyboard
2.1.4 Character Key Shortcuts - If shortcuts exist, they can be turned off or remapped
2.2.1 Timing Adjustable - Time limits can be adjusted or extended
2.2.2 Pause, Stop, Hide - Provide controls for moving content
2.3.1 Three Flashes - No content flashes more than three times per second
2.4.1 Bypass Blocks - Provide skip links
2.4.2 Page Titled - Pages have descriptive titles
2.4.3 Focus Order - Focus order preserves meaning
2.4.4 Link Purpose (In Context) - Link purpose is clear from context
2.5.1 Pointer Gestures - Complex gestures have simple alternatives
2.5.2 Pointer Cancellation - Actions completed on up-event
2.5.3 Label in Name - Visual labels match accessible names
2.5.4 Motion Actuation - Functions triggered by motion can be operated by UI
3.1.1 Language of Page - Page language is programmatically set
3.2.1 On Focus - Elements don't change on focus
3.2.2 On Input - Elements don't change on input
3.3.1 Error Identification - Errors are clearly identified
3.3.2 Labels or Instructions - Form elements have labels
4.1.1 Parsing - Content uses valid markup
4.1.2 Name, Role, Value - UI components have proper roles and states

Level AA (Should Have):
1.2.4 Captions (Live) - Live videos have captions
1.2.5 Audio Description - Videos have audio descriptions
1.3.4 Orientation - Content not restricted to orientation
1.3.5 Identify Input Purpose - Input fields have proper purpose
1.4.3 Contrast (Minimum) - Text has sufficient contrast
1.4.4 Resize Text - Text can be resized without loss of functionality
1.4.5 Images of Text - Use text instead of images of text
1.4.10 Reflow - Content reflows on zoom
1.4.11 Non-text Contrast - UI elements have sufficient contrast
1.4.12 Text Spacing - Text can be adjusted without loss of functionality
1.4.13 Content on Hover or Focus - Additional content can be dismissed
2.4.5 Multiple Ways - Multiple ways to find pages
2.4.6 Headings and Labels - Headings and labels are descriptive
2.4.7 Focus Visible - Keyboard focus is visible
3.1.2 Language of Parts - Language changes are indicated
3.2.3 Consistent Navigation - Navigation is consistent
3.2.4 Consistent Identification - Components are identified consistently
3.3.3 Error Suggestion - Error suggestions are provided
3.3.4 Error Prevention - Critical forms can be reviewed
4.1.3 Status Messages - Status messages can be programmatically determined

Level AAA (Nice to Have):
1.2.6 Sign Language - Videos have sign language interpretation
1.2.7 Extended Audio Description - Videos have extended audio description
1.2.8 Media Alternative - Alternative for time-based media
1.2.9 Audio-only (Live) - Alternative for live audio
1.3.6 Identify Purpose - Purpose of UI components can be programmatically determined
1.4.6 Enhanced Contrast - Enhanced contrast for text
1.4.7 Low or No Background Audio - Audio is clear from background
1.4.8 Visual Presentation - Text presentation can be customized
1.4.9 Images of Text (No Exception) - No images of text
2.1.3 Keyboard (No Exception) - All functionality available from keyboard
2.2.3 No Timing - No time limits
2.2.4 Interruptions - Interruptions can be postponed
2.2.5 Re-authenticating - Data preserved for re-authentication
2.2.6 Timeouts - Users warned of timeouts
2.3.2 Three Flashes (No Exception) - No flashing content
2.3.3 Animation from Interactions - Motion animation can be disabled
2.4.8 Location - Location information available
2.4.9 Link Purpose (Link Only) - Link purpose clear from link text
2.4.10 Section Headings - Sections have headings
2.5.5 Target Size - Click targets are large enough
2.5.6 Concurrent Input Mechanisms - Multiple input methods available
3.1.3 Unusual Words - Unusual words are explained
3.1.4 Abbreviations - Abbreviations are explained
3.1.5 Reading Level - Content is understandable
3.1.6 Pronunciation - Pronunciation is provided
3.2.5 Change on Request - Changes occur only on request
3.3.5 Help - Context-sensitive help is available
3.3.6 Error Prevention (All) - All forms can be checked and corrected
